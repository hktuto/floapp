# 擴展功能規格 (v2.0)

## 概述
將 Flo App 從個人記錄應用擴展為運動課程預約平台，支持多角色（Admin/Teacher/User）同代幣（Token）經濟系統。

---

## 功能 1: Avatar 頭像支持

### 目的
測試 Nuxt Hub Blob 附件功能，同時提升用戶體驗。

### 需求
- [ ] 用戶可以上傳頭像圖片
- [ ] 支持圖片裁切功能
- [ ] 頭像顯示於 Navbar、Settings、Friends 列表

### 數據庫變更
```sql
-- user_profiles 表
avatar_url: text (nullable) -- Blob 存儲路徑
```

### API 端點
- `POST /api/avatar/upload` - 上傳頭像到 Nuxt Hub Blob
- `DELETE /api/avatar/delete` - 刪除頭像

### 前端改動
- Settings 頁面添加頭像上傳區域
- Navbar 右上角顯示用戶頭像
- Friends 列表顯示頭像縮略圖

## 功能 2: 角色系統 (RBAC)

### 角色定義
| 角色 | 權限 | 說明 |
|------|------|------|
| **admin** | 系統管理、用戶角色設置、token 管理、全局課程管理 | 平台管理員 |
| **teacher** | 創建/管理課程、管理學生、查看預約 | 教練/導師 |
| **user** | 預約課程、查看記錄、購買 token | 普通用戶/學生 |

### 重要規則
- **只有 admin 可以修改用戶角色**
- User 無法自行申請成為 teacher
- Admin 通過 Admin Panel 指派角色

### 數據庫變更
```sql
-- user_profiles 表添加
description: text (支持 markdown) -- 用戶自我介紹
role: enum('admin', 'teacher', 'user') default 'user'
```

### 權限控制
- Middleware: `role.global.ts` 檢查角色權限
- 頁面保護：
  - `/admin/*` - Admin only
  - `/teacher/*` - Teacher only
  - `/courses/book` - User/Teacher (可預約)

## 功能 3: Token 代幣經濟系統

### 概念
- 平台使用 **Token** 作為課程交易貨幣
- 1 個 Token = 1 個課程名額（或根據課程設置）
- Admin/Teacher 創建 Token Package，用戶購買後獲得 Token

### 數據庫設計

#### token_packages (代幣套餐)
```sql
- id, name
- description: text (markdown) -- 套餐介紹
- token_amount: integer -- 包含多少 token
- price_hkd: decimal -- 參考價格（港幣）
- status: enum('active', 'inactive')
- created_by: admin_id 或 teacher_id
- created_at, updated_at
```

#### user_tokens (用戶代幣餘額)
```sql
- user_id (PK)
- balance: integer -- 剩餘 token 數
- total_earned: integer -- 累計獲得
- total_spent: integer -- 累計使用
- updated_at
```

#### token_transactions (代幣交易記錄)
```sql
- id, user_id
- type: enum('purchase', 'booking', 'refund', 'admin_grant', 'admin_deduct')
- amount: integer (+/-)
- description: text
- related_package_id -- 如果係購買套餐
- related_booking_id -- 如果係課程相關
- created_by -- 操作人（admin 或系統）
- created_at
```

### 流程
```
購買 Token:
User → 瀏覽 Token Packages → 選擇套餐 → 購買 → 餘額增加

使用 Token:
User → 瀏覽課程 → 預約 → 扣除 Token → 生成 Booking

Admin 操作:
Admin → 用戶管理 → 充值/扣除 Token → 記錄交易
```

## 功能 4: 運動課程預約系統

### 核心概念: Lazy Instance 生成
- 課程模板設置**重複規則**（例如每週一三五 10:00）
- Frontend 計算所有可能的時段並顯示
- **只有當用戶預約時，才生成實際的 course_instance 記錄**
- 好處：數據庫輕量，避免大量空白記錄

### 數據庫設計

#### courses (課程模板)
```sql
- id, teacher_id
- title
- description: text (markdown) -- 課程詳細介紹
- category: enum('yoga', 'pilates', 'boxing', 'swimming', 'hiit', 'dance', etc.)
- duration_minutes: integer
- token_cost: integer -- 課程所需 token 數
- max_students: integer
- location_type: enum('online', 'offline')
- location_address: text -- 線下地址或線上鏈接
- status: enum('active', 'inactive')
- is_recurring: boolean -- 是否重複課程
- recurring_pattern: jsonb -- 重複規則
  -- 例如: {days: [1,3,5], time: "10:00", start_date: "2025-04-01", end_date: "2025-06-30"}
- created_at, updated_at
```

#### course_instances (課程實例 - Lazy 生成)
```sql
- id, course_id
- scheduled_at: timestamp -- 具體上課時間
- status: enum('open', 'full', 'cancelled', 'completed')
- booked_count: integer -- 已預約人數
- created_at -- 首次有人預約時生成
```

#### course_bookings (預約記錄)
```sql
- id, instance_id, user_id
- token_amount: integer -- 實際使用的 token
- status: enum('booked', 'cancelled', 'completed', 'no_show')
- booked_at, cancelled_at
- note: text -- 用戶備註
```

#### course_students (課程學生名單 - Teacher 管理)
```sql
- id, course_id, user_id
- enrolled_by: user_id -- 添加人（teacher 或自己）
- status: enum('active', 'removed', 'completed')
- note: text -- 老師備註
- created_at, removed_at
```

### 重複規則設計 (recurring_pattern)
```json
{
  "type": "weekly",
  "days": [1, 3, 5],      // 週一、三、五
  "time": "10:00",         // 10:00 AM
  "start_date": "2025-04-01",
  "end_date": "2025-06-30",
  "exclude_dates": [       // 跳過的日期（假期等）
    "2025-04-18",
    "2025-05-01"
  ]
}
```

### 預約流程
```
1. User 瀏覽 /courses
   └── 顯示所有課程模板

2. User 查看課程詳情 /courses/[id]
   └── Frontend 根據 recurring_pattern 計算未來 30 日的所有時段
   └── 顯示為「可預約」列表

3. User 點擊預約
   ├── 檢查：token 餘額是否足匱？
   ├── 檢查：是否已達 max_students？
   ├── 扣除 token
   ├── 生成 course_instance（如果不存在）
   ├── 創建 course_booking 記錄
   └── 返回預約確認

4. Teacher 查看 /teacher/schedules
   └── 日曆視圖顯示所有已預約的課程
```

## 技術實現要點

### 1. Markdown 支持
- 使用 `@nuxtjs/mdc` 或 `marked` 渲染 markdown
- 編輯器：簡單 textarea 或 WYSIWYG (TipTap)

### 2. 圖片處理
- 使用 `vue-advanced-cropper` 做頭像裁切
- Nuxt Hub Blob 存儲圖片

### 3. 日曆組件
- 使用 `@nuxt/ui` 的 Calendar 或 `vue-cal`
- 顯示預約事件、課程時間

### 4. Lazy Instance 生成
- Frontend 使用 date-fns 計算重複日期
- API 端點只在預約時創建 instance 記錄
- 檢查 instance 是否已存在（避免重複創建）

### 5. Token 交易原子性
- 使用數據庫事務確保：
  - 扣除 token
  - 創建 booking
  - 兩者同時成功或同時失敗

## API 端點規劃（新增）

### Avatar
- `POST /api/avatar/upload` - 上傳頭像
- `DELETE /api/avatar/delete` - 刪除頭像

### Admin
- `GET /api/admin/users` - 用戶列表
- `PUT /api/admin/users/:id/role` - 修改角色
- `POST /api/admin/users/:id/tokens` - 充值/扣除 token

### Token Packages
- `GET /api/token-packages` - 套餐列表
- `POST /api/token-packages` - 創建套餐 (Admin/Teacher)
- `POST /api/token-packages/:id/purchase` - 購買套餐

### User Tokens
- `GET /api/my-tokens` - 我的餘額同交易記錄

### Courses
- `GET /api/courses` - 課程列表（分類篩選）
- `GET /api/courses/:id` - 課程詳情
- `GET /api/courses/:id/available-slots` - 可預約時段
- `POST /api/courses` - 創建課程 (Teacher)
- `PUT /api/courses/:id` - 編輯課程 (Teacher)

### Bookings
- `POST /api/bookings` - 預約課程（生成 instance）
- `DELETE /api/bookings/:id` - 取消預約
- `GET /api/my-bookings` - 我的預約

### Teacher
- `GET /api/teacher/courses` - 我的課程
- `GET /api/teacher/schedules` - 我的時間表
- `GET /api/teacher/courses/:id/students` - 課程學生
- `POST /api/teacher/courses/:id/students` - 添加學生
- `DELETE /api/teacher/courses/:id/students/:userId` - 移除學生
